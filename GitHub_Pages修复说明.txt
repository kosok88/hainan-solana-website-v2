## 🚀 完整修复步骤

### 第1步：确认 GitHub Pages 设置

```
1. 打开 GitHub 仓库
   https://github.com/kosok88/hainan-solana-website-v2

2. 点击 Settings（设置）

3. 左侧菜单找到 "Pages"

4. 查看部署来源：
   - Source: 应该是 "Deploy from a branch"
   - Branch: 应该选择 "main" 或 "master"
   - Folder: 应该选择 "/ (root)"
```

### 第2步：上传所有修复文件

**需要上传的文件：**

```
✅ index.html（导航链接已修复）
✅ hainan-videos.html（新视频页面）
✅ 所有其他已修复的文件
```

**上传步骤：**

```
1. 打开 GitHub 仓库主页

2. 确认你在正确的分支（main 或 master）

3. 上传/替换以下文件：
   - index.html（修复后的版本）
   - hainan-videos.html（新文件）
   
4. 如果需要，删除：
   - videos.html（旧文件，Rick的地盘）

5. 提交所有更改
```

### 第3步：等待 GitHub Pages 重新部署

```
1. 提交后，GitHub Pages 会自动重新构建

2. 等待时间：2-5 分钟

3. 可以在 Settings → Pages 查看部署状态
   - 绿色勾：部署成功
   - 黄色圈：正在部署
   - 红色叉：部署失败
```

### 第4步：清除浏览器缓存

```
因为你之前访问过旧版本，需要清除缓存：

方法1：硬刷新
Windows: Ctrl + Shift + R
Mac: Command + Shift + R

方法2：无痕模式
Chrome: Ctrl + Shift + N
Firefox: Ctrl + Shift + P

方法3：清除所有缓存
浏览器设置 → 清除浏览数据 → 缓存的图片和文件
```

## 🔍 如果还有问题

### 问题1：GitHub Pages 没有自动部署

**解决方法：**
```
1. 进入 Settings → Pages
2. 临时改变 Source 为 "None"
3. 保存
4. 再改回 "Deploy from a branch"
5. 选择分支（main）
6. 保存
7. 触发重新部署
```

### 问题2：部署了但还是旧内容

**原因：** GitHub Pages 有CDN缓存

**解决方法：**
```
1. 等待 10-15 分钟（CDN缓存过期）
2. 或使用查询参数强制刷新：
   https://kosok88.github.io/hainan-solana-website-v2/?v=2
```

### 问题3：404 错误

**原因：** 文件路径不对

**检查：**
```
1. 文件名是否正确（hainan-videos.html）
2. 文件是否在根目录
3. 分支是否正确
```
